# Code Quality Control

## کنترل کیفیت کد

کیفیت کدهای نوشته‌شده‌ی توسعه دهنده از جنبه‌های متفاوت در سه سطح بررسی می‌شود.

---

## Beginner

### مبتدی

- [ ] رعایت کنوانسیون‌های نام‌گذاری Naming Convention، Style Guide
- [ ] حذف کد کامنت‌شده غیرضروری
- [ ] وجود کامنت‌های مهم و به‌روز
- [ ] ساختار منظم فایل و پوشه
- [ ] استفاده از استاندارد کدنویسی تیمی (StyleCop, Prettier و …)
- [ ] متدهای کوتاه با مسئولیت مشخص (پیروی از اصل SRP)
- [ ] جداسازی Concernها در کد
- [ ] کامیت های معنادار نوشته شود
- [ ] نوشتن hardcode مجاز نمی باشد و به هیچ عنوان پذیرفته نیست
- [ ] اصل DRY (عدم تکرار کد)

---

## Intermediate

### متوسط

- [ ] نوشتن تست کیس برای کدها

- [ ] نام‌گذاری منطقی کلاس و متد
